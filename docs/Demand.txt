# **一、学生用户核心需求**

1.  **个人信息与偏好录入**：学生用户需要能够便捷地填写并提交包含个人多维度信息的问卷。这些维度应包括但不限于MBTI人格类型（或通过系统内简版IPIP人格量表测试生成）、作息时间习惯（如早睡早起型、夜猫子型）、个人卫生清洁习惯（如频率、标准）、学习习惯（如是否需要在宿舍学习、安静度要求）、社交偏好（如内向/外向、喜欢独处/集体活动）、兴趣爱好（如音乐偏好、运动类型）等。系统需要能清晰引导学生完成问卷填写。**（maybe可以和一些问卷APP进行链接等等）（根据查找到的文献，最好可以依据五大人格类型进行匹配，稳定性更好）**

4.  **个性化心理健康资源获取**：学生用户可以根据其匹配结果或在宿舍生活中遇到的潜在问题（如通过系统预警间接感知），接收到系统推送的个性化心理健康课程或建议，例如“冲突解决技巧”、“有效沟通”等，以提升其社会适应能力。**（可以链接一些心理网站公众号等等）**

6.**入住后反馈与调换申请**：在入住一段时间后，学生用户可以通过系统匿名或实名提交关于宿舍匹配满意度的反馈。当出现严重且无法调和的矛盾时，学生可以通过此系统提交调换宿舍的申请，并说明原因，申请将流转至管理员进行审批。

# **二、高校管理员用户核心需求与用例**

1.  **学生数据与匹配管理**：高校管理员需要一个管理平台，能够查看和管理学生提交的个人信息及问卷数据（需有严格的隐私保护措施）。管理员应能启动和监控整个宿舍匹配流程，并能查看最终的匹配结果概览。

2.  **矛盾预警与干预支持**：系统需要为管理员提供宿舍矛盾预警报告功能。该报告应基于对学生投诉数据（若能接入）、系统内关键词监控（如学生反馈中的负面词汇）以及投诉频次等进行分析，帮助管理员提前识别潜在的宿舍矛盾，并及时介入处理。

3.  **匹配规则与权重调整（可选，但建议）**：管理员可能需要根据学校的实际情况和特定时期的需求，对匹配维度及其权重分配逻辑进行一定的调整和配置，以优化匹配效果。

4.  **宿舍文化建设支持**：管理员能够通过平台获取或发布宿舍文化活动的策划模板和案例（如“共读计划”、“清洁挑战赛”），以引导和鼓励学生积极参与宿舍文化建设，营造和谐的宿舍氛围。（发布活动？是不是要再搞一套，难度如何？）

6.**宿舍资源与房态管理**：管理员需要一个可视化的宿舍楼宇及房间管理模块，能够清晰查看各宿舍楼的床位总数、已分配数、剩余数、房间类型（如四人间、六人间）、性别的状态。

7. **数据洞察与决策支持报告**：系统应能定期生成多维度的数据分析报告，例如：不同学院/专业学生的作息习惯分布、整体人格类型概览、宿舍矛盾高发区域或群体的特征分析等。这能为学校的学生管理工作和资源配置提供长期的决策支持。

# **三、系统自身核心需求与功能**

1.  **多维度数据量化与处理**：系统必须能够将收集到的学生心理学维度信息（如性格、作息、社交偏好等）进行有效量化，并建立相应的数据库进行存储和管理。系统需要具备数据清洗能力，以保证数据质量。

2.  **AI匹配算法实现**：系统核心是其AI匹配引擎，需要实现基于机器学习模型（如协同过滤、聚类分析）的匹配算法。该算法能够综合考虑多个匹配维度及其权重，计算学生间的匹配度。

8. **自动化通知与提醒系统**：系统应具备自动化的消息推送功能，能在关键时间节点向学生和管理员发送通知。例如：提醒学生及时填写问卷、通知分配结果已发布、告知管理员有新的调宿申请待处理等，以提升系统易用性和管理效率。
